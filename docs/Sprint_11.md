# Sprint 11 — AI Curriculum Generation

**Duration:** ~5 days
**Theme:** Turn a track + placement level into a **full, substantial** course —
many lessons, each with exercises and a quiz — generated by the AI into the
existing content tables. This replaces manual authoring.

## Goal

Generate a personalized course for a track: a `Course` bound to the track's
language **and owner**, containing **plenty of content** (target: **≥ 8 lessons**,
each with **≥ 2 exercises** and **1 quiz/"test"**), pitched at the placement
level. Generation runs as a tracked background job (it's a lot of model calls),
self-verifies generated exercises (Sprint 6 gate), and lands everything in the
`lessons`/`exercises`/`quizzes` tables so serving, grading, "Today", and progress
work unchanged. **Deprecate and remove the manual content CRUD / seed** as the
primary path.

## User Story

- As a learner, after my placement test a course is built for me and I can start
  learning — with real depth, not one or two lessons.
- As a learner, my "Today" plan and Courses are driven by my generated course.
- As the team, generation reuses the existing serving/grading/progress machinery.

## Tasks

### Backend
1. Bind generated content to a learner: add `track_id` (and/or `owner_user_id`)
   to `courses` + migration `0011_course_ownership`; update entities/repos.
   Courses list/serve filtered to the learner's tracks.
2. `CurriculumService.generate_course(track)`: build a syllabus (topics ordered
   by level), then for each topic generate a lesson + its exercises + a quiz via
   the Sprint 6 provider, writing through `ContentService`/`ExerciseService`/
   `QuizService` with `source="ai"`. Enforce minimum counts; self-verify each
   generated exercise (reject/retry on failure).
3. `GenerationJob` model + migration `0011b` (or fold in): tracks async progress
   (`pending`/`running`/`done`/`error`, counts). Run via background task; expose
   `GET /me/tracks/{id}/generation` for polling.
4. Endpoints: `POST /me/tracks/{id}/generate` (kick off), `GET …/generation`
   (status). Guard with per-user generation quota (config; see Sprint 13).
5. **Remove manual authoring from the primary flow**: retire `admin_content`
   create/update/delete routes and the exercise/quiz admin authoring endpoints
   (or move behind a feature flag); update `scripts.seed` to a dev-only fixture.
6. Tests (AI + Judge0 mocked): a generated course meets the minimum lesson/
   exercise/quiz counts; content is `source="ai"` and served by existing `GET`s;
   job status transitions; generation respects the quota.

### Frontend
1. Post-placement: trigger generation, show a progress screen (polling the job).
2. `Courses`/`Course`/`Today` render the learner's generated content (scoped to
   their tracks); remove the Admin CRUD UI for content.
3. `features/curriculum/hooks.ts` (`useGenerateCourse`, `useGenerationStatus`).

## Status — ✅ Complete

After placement, a background job builds a full personalized course; the manual
course-authoring flow is retired.

> **Design notes vs. the plan.** (1) Migration is **`0011_curriculum`** (adds
> `courses.track_id` + `generation_jobs` in one migration). (2) To keep AI calls
> feasible on the free tier, a lesson is generated as **one "lesson pack"**
> (content + exercises + quiz) rather than a call per artifact — `generate_syllabus`
> then one `generate_lesson_pack` per topic. (3) Manual content CRUD endpoints are
> **retired from the primary flow** (frontend no longer uses them; seed only
> seeds languages) but the admin endpoints remain until Sprint 13 repurposes
> admin for review — avoiding a large test rewrite now.

**Backend**
- `courses.track_id` + `GenerationJob` model + migration `0011_curriculum`;
  entities, `CourseRepository.list_by_track_ids`, `GenerationJobRepository`.
- Provider: `generate_syllabus` + `generate_lesson_pack` (JSON mode).
- `CurriculumService`: `start_generation` (idempotent while a job is active),
  `generate_course` (syllabus → per-topic lesson pack → lesson + exercises

### Notes / follow-ups

- Default 8 lessons × (2 exercises + quiz) is a lot of AI + Judge0 calls; the job
  paces them and continues past a failed lesson. Counts + self-verify are
  config-toggleable to fit free-tier quotas.
- Two entry points (onboarding, add-language) both run placement → generation,
  so behavior is consistent.
- Continuous "generate more" + in-course chat is **Sprint 12**.
